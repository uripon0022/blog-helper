# WordPress + AFFINGER 投稿ヘルパー 要件定義

## 1. ツールの目的

WordPress で運用している楽天アフィリエイト向けブログの記事作成と投稿作業を効率化するための専用ツールを作成する。

対象は `WordPress + AFFINGER` 環境とし、記事作成、保存、転送、投稿前確認までを支援する。

完全自動公開は行わず、WordPress 管理画面に記事内容を転送し、公開前にユーザーが内容を確認できる設計にする。

---

## 2. 基本方針

ツール上で記事を作成、保存、管理し、WordPress の投稿編集画面へ内容を自動入力する。

最終的な公開は自動化せず、以下のいずれかで止める。

- WordPress 側で下書き保存まで自動実行する
- 下書き保存の直前まで入力し、ユーザーが保存ボタンを押す

安全性を優先し、以下は行わない。

- 自動で公開ボタンを押す
- ユーザー確認なしで記事を公開する
- 複数記事を連続で自動公開する

---

## 3. 想定する利用の流れ

1. ツール上で記事タイトルや本文を入力する
2. 必要に応じてカテゴリ、タグ、メタディスクリプション、PR表記、定型文などを設定する
3. ツール内に記事を保存する
4. 「WordPressへ転送」ボタンを押す
5. WordPress の投稿編集画面を開く
6. 各入力欄へ内容を自動入力する
7. 下書き保存する、または保存直前で止める
8. ユーザーが WordPress 側で内容を確認する
9. ユーザーが手動で公開する

---

## 4. 対象環境

初期版の対象は以下とする。

- WordPress 管理画面
- AFFINGER を利用した投稿画面
- 楽天アフィリエイト記事の作成運用

補足:

- メタディスクリプションは WordPress 標準機能ではなく、AFFINGER 側の投稿設定欄に入力する想定とする
- 将来的に Yoast SEO、Rank Math、All in One SEO などへの対応を追加できるように、SEO入力欄の処理は分離して実装する

## 6. 記事保存機能

ツール内で作成した記事を保存できるようにする。

最低限、以下の状態を管理する。

- 作成中
- 未転送
- 転送済み
- 公開済み
- 修正予定

記事一覧から過去の記事を開いて再編集できるようにする。

記事の複製機能も用意する。

同じ構成の記事を量産する可能性があるため、過去記事を複製して新しい記事として編集できるようにする。

---

## 7. WordPress への転送機能

ブラウザ拡張機能などを利用し、WordPress の投稿編集画面に以下の内容を自動入力する。

- 記事タイトル
- 記事本文
- カテゴリ
- タグ
- 投稿日時
- 抜粋
- メタディスクリプション

AFFINGER の投稿画面に専用入力欄がある項目については、その欄へ直接入力する。

対象項目の優先順位は以下とする。

1. 記事タイトル
2. 記事本文
3. メタディスクリプション
4. カテゴリ
5. タグ
6. 抜粋
7. 投稿日時

---

## 8. メタディスクリプション対応

AFFINGER 環境では記事ごとにメタディスクリプションを入力できる前提で、初期版から対応対象に含める。

### ツール側の仕様

- メタディスクリプション入力欄を用意する
- 記事データとして保存する
- テンプレートに含められるようにする
- 複製時にも引き継げるようにする

### 転送時の仕様

- WordPress 投稿画面内の AFFINGER 用メタディスクリプション欄を探して入力する
- 入力欄が見つからない場合は警告を表示する
- メタディスクリプション欄が見つからなくても、タイトルと本文の転送は継続してよい

### 補足

- 将来的に SEO プラグイン別の入力欄へ切り替えられるよう、メタディスクリプション入力処理はモジュール化する

---

## 9. 転送後の動作

「WordPressへ転送」を押した後は、投稿編集画面を開き、自動入力を実行する。

その後は以下の優先順位で動作させる。

### 第一希望

内容を自動入力し、WordPress 側で下書き保存まで行う。

保存後は、その記事の編集画面を表示したままにする。

### 第二希望

内容の自動入力まで行い、下書き保存ボタンを押す直前で止める。

ユーザーが内容を確認し、自分で下書き保存ボタンを押す。

### やらないこと

- 自動で公開ボタンを押す
- 予約投稿を自動確定する
- 既存記事をユーザー確認なしで上書きする

---

## 10. 楽天アフィリエイト向け機能

楽天アフィリエイト記事の作成を想定し、以下の機能を持たせる。

- 楽天アフィリエイトリンクの保存
- 商品名の保存
- 商品紹介用の定型文
- PR表記の自動挿入
- 記事冒頭への PR 表記挿入
- 記事末尾への定型文挿入
- よく使うリンクの登録
- よく使う文章テンプレートの登録
- 商品ごとの記事複製

初期版では、リンク、商品名、PR表記、定型文の保存と本文挿入まで対応すればよい。

## 11. テンプレート機能

記事の型をテンプレートとして保存できるようにする。

例:

- 商品紹介記事
- 口コミ記事
- 比較記事
- セール情報記事
- 実際に使った感想記事

テンプレートには以下を含められるようにする。

- 見出し構成
- 記事冒頭文
- PR表記
- 記事末尾文
- 定型リンク
- 注意書き
- メタディスクリプションの型
- 抜粋の型

テンプレートを選ぶと、新規記事に内容が反映されるようにする。

## 12. プレビュー機能

WordPress へ転送する前に、ツール上で記事の見た目を確認できるようにする。

最低限、以下を確認できるようにする。

- タイトル
- 本文
- 改行
- 見出し
- リンク
- 画像プレースホルダ
- PR表記
- 定型文
- メタディスクリプション
- 抜粋

WordPress 上での完全な表示再現が難しい場合、簡易プレビューで問題ない。

## 13. 転送前チェック

転送時に、以下の不足やミスをチェックする。

- タイトルが空欄
- 本文が空欄
- PR表記が必要なのに入っていない
- 楽天アフィリエイトリンクが未入力
- カテゴリが未選択
- タグが未設定
- メタディスクリプションが未入力
- 抜粋が未入力
- 本文内に未置換のテンプレート文字が残っている

警告は表示するが、必ずしも転送を禁止しなくてよい。

---

## 14. エラー時の動作

WordPress の投稿編集画面が開けない、または入力欄が見つからない場合は処理を中断する。

その際、以下のような分かりやすいエラーを表示する。

- WordPress にログインしてください
- 投稿画面を開けませんでした
- タイトル入力欄が見つかりませんでした
- 本文入力欄が見つかりませんでした
- メタディスクリプション入力欄が見つかりませんでした
- AFFINGER の画面仕様が変更された可能性があります
- 下書き保存に失敗しました

記事データはツール側に残し、エラーによって消えないようにする。

---

## 15. データ保存

記事やテンプレートはブラウザ内に保存する。

ただし、ブラウザの履歴削除やサイトデータ削除で消える可能性があるため、バックアップ機能を付ける。

必要な機能は以下。

- 全データを JSON 形式で書き出す
- JSON ファイルから復元する
- バックアップ時に注意文を表示する
- 保存データが消える可能性をツール内に明記する

将来的にはクラウド保存も検討できるが、初期版ではローカル保存でよい。

## 18. 技術的な注意点

WordPress に正式な投稿 API を使わない場合は、ブラウザ拡張機能による自動入力方式を採用する。

WordPress のブロックエディタやクラシックエディタ、AFFINGER 独自設定欄の構造を確認したうえで入力処理を実装する。

単純に input や textarea へ value を設定するだけでは、WordPress 側が入力を認識しない可能性がある。

そのため、必要に応じて以下のイベントを発火させる。

- input
- change
- blur
- keyup

Gutenberg の場合は React ベースの状態管理を考慮し、必要なら `dispatchEvent` やエディタ専用 API の利用も検討する。

ログイン情報やパスワードはツール内に保存しない。

WordPress へのログインは、ユーザーが通常どおりブラウザ上で行う。

---

## 19. 実装上の分離方針

画面仕様変更に対応しやすくするため、サイト固有処理を分離する。

最低限、以下を分けて実装する。

- WordPress 共通入力処理
- Gutenberg 用本文入力処理
- クラシックエディタ用本文入力処理
- AFFINGER 用メタディスクリプション入力処理
- カテゴリ、タグ、日時入力処理
- 下書き保存処理

将来的に別テーマや別 SEO プラグインへ対応しやすい構成にする。

---

## 20. 完成イメージ

ツール上で楽天アフィリエイト用の記事を作成し、「WordPressへ転送」を押す。

WordPress の投稿編集画面が開き、タイトル、本文、メタディスクリプション、必要に応じてカテゴリやタグが自動入力される。

可能であれば下書き保存まで行い、保存後の編集画面を開く。

ユーザーは改行、装飾、リンク、画像、PR表記、抜粋、メタディスクリプションを確認し、問題がなければ自分で公開する。

公開まで完全自動にはせず、必ずユーザーの最終確認を挟む。
